= 使用 SR-IOV 和 NFS 在 SUSE KVM 上部署 SAP HANA，并使用NetApp存储。
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
使用带有 SR-IOV 网络接口和 NFS 或 FCP 存储访问的NetApp存储，在 SUSE KVM 上部署 SAP HANA 单主机。按照此工作流程配置虚拟接口，将其分配给虚拟机，并设置存储连接以获得最佳性能。

有关 SAP HANA on KVM 虚拟化的概述，请参阅 SUSE 文档： https://documentation.suse.com/sbp/sap-15/pdf/SBP-SLES4SAP-HANAonKVM-SLES15SP5_en.pdf["SUSE 在 KVM 上部署 SAP HANA 的最佳实践"] 。

image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["一个"]link:kvm-hana-requirements.html["查看配置要求"]

[role="quick-margin-para"]
审查在 SUSE KVM 上使用NetApp存储、SR-IOV 和存储协议部署 SAP HANA 的关键要求。

image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["两个"]link:kvm-hana-network.html["配置SR-IOV网络接口"]

[role="quick-margin-para"]
在 KVM 主机上设置 SR-IOV（单根 I/O 虚拟化），并为 VM 分配虚拟接口以进行网络通信和存储访问。

image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["三"]link:kvm-hana-fcn.html["配置光纤通道网络"]

[role="quick-margin-para"]
将物理 FCP HBA 端口分配给 VM 作为 PCI 设备，以便在 SAP HANA 中使用 FCP LUN。

image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["四个"]link:kvm-hana-storage.html["配置用于 SAP HANA 的NetApp存储"]

[role="quick-margin-para"]
在虚拟机和NetApp存储系统之间建立 NFS 或 FCP 存储连接，用于 SAP HANA 数据库文件。
